## 2. 상세 분석

### 2.1 AI 비서 서비스

#### ISP 근거
```
"AI·모바일 기반 의정지원 통합"
"DID(모바일 신분증) + AI 비서 서비스"

출처: ("Goal Setting", pp. 21-23)
```

```
"내부망/외부망 구조에서 AI 비서가 포함된 협업/업무체계"

출처: ("Goal Setting", pp. 167-168)
```

#### 제안 내용
- OpenClaw를 활용한 AI 비서 에이전트
- 의정 업무 지원 (조례 검토, 회의록 요약, 일정 관리)
- 내부망에서 안전하게 운영

#### 정합성: ✅ 직접 부합
ISP에서 AI 비서 서비스를 명시적으로 언급하고 있으며, 제안 내용이 이를 구현합니다.

### 2.2 내부망 LLM + RAG

#### ISP 근거
```
"LLM + RAG(검색증강생성)로 외부 지식베이스 결합"
"회의록 데이터 구조화 및 LLM/RAG 구성"

출처: ("Goal Setting", pp. 285-288)
```

```
"조례안 분석에 RAG 구성이 필요"

출처: ("Goal Setting", p. 120)
```

#### 제안 내용
- 내부망에 LLM 서버 운영 (기존 인프라 활용)
- RAG 시스템으로 의정자료 검색 (회의록, 조례, 의안)
- 벡터 DB 기반 시맨틱 검색

#### 정합성: ✅ 직접 부합
ISP에서 LLM/RAG 기반 지식결합을 명시적으로 요구하고 있습니다.

### 2.3 DID 기반 인증

#### ISP 근거
```
"블록체인(DID) 기반 모바일 신분증 + AI 적용"
"모바일 신분증(DID) + AI 비서 서비스 방향"

출처: ("Goal Setting", pp. 21, 23)
```

```
"의원 전용 기능 권한 설정과 인증"

출처: ("Goal Setting", p. 128)
```

#### 제안 내용
- DID 기반 모바일 신분증 인증
- 2FA(OTP) 추가 인증
- 역할별 권한 제어 (RBAC)

#### 정합성: ✅ 직접 부합
ISP에서 DID 기반 인증을 명시적으로 요구하고 있습니다.

---

### 2.4 외부 접근 (VPN/2FA)

#### ISP 근거
```
"모바일/외부 PC에서 내부망 포털 접근을 위한 VPN·2단계 인증 + 권한 설정"

출처: ("Goal Setting", p. 128)
```

```
"내부망 ↔ 인터넷망 간 '내외부 동기화 기반' 연계 개념"

출처: ("Goal Setting", p. 177)
```

#### 제안 내용
- VPN 터널을 통한 암호화 접속
- 2FA(DID + OTP) 인증
- VDI 프로토콜로 화면만 전송 (데이터 반출 차단)

#### 정합성: ✅ 직접 부합
ISP에서 외부 접근 시 VPN + 2FA를 명시적으로 요구하고 있습니다.

---

### 2.5 클라우드 기반 인프라

#### ISP 근거
```
"클라우드 기반 통합 인프라 구축"

출처: ("Goal Setting", p. 16)
```

#### 제안 내용
- 옵션 A: 온프레미스 VDI
- 옵션 B: 프라이빗 클라우드 VDI
- 옵션 C: 하이브리드

#### 정합성: ✅ 방향 부합
ISP에서 클라우드 기반 인프라를 언급하고 있으며, 옵션 B/C가 이에 해당합니다.
옵션 A도 가상화 기반이므로 방향에 부합합니다.

---

### 2.6 VDI 환경

#### ISP 근거
```
ISP에 'VDI'라는 용어가 직접 등장하지는 않음.
그러나 "클라우드 기반 통합 인프라 구축" 목표의 구현 수단으로 VDI는 자연스러운 선택.

출처: ("Goal Setting", p. 16)
```

#### 제안 내용
- VDI를 통한 통합 업무환경
- 의원 1명당 5개 PC → 1개 VDI 세션
- 어디서나 동일한 업무환경

#### 정합성: ⚖️ 구현 옵션
VDI는 ISP에 직접 언급되지 않았으나, ISP 목표를 달성하기 위한 기술적 구현 방법입니다.
