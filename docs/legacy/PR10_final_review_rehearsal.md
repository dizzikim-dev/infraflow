**아젠다 공유**:
```
"오늘 미팅은 약 1시간 진행될 예정입니다.

2. 저희가 준비한 VDI + AI 비서 제안을 발표한 뒤,
3. 질의응답과 다음 단계 협의로 마무리하겠습니다.

진행 중 궁금하신 점은 편하게 질문해 주세요."
```

#### [2] 현황 확인 (10분)

**질문 유도**:
```
"제안에 앞서 몇 가지 현황을 확인드리고자 합니다.

첫째, 현재 의원분들이 사용하시는 PC 환경은 어떻게 되나요?
저희가 알기로 본회의장, 상임위, 의원실, 지역상담소, 노트북
등 5개 환경을 사용하시는 것으로 알고 있는데 맞을까요?"
```

**핵심 질문 5개** (시간 내 가능한 만큼):
1. 내부망 LLM 현황 (위치, 모델, GPU)?
2. 외부 접근 정책 (VPN, 2FA, DID)?
3. VDI 보안 수준 (클립보드, 파일반출)?
4. AI가 접근할 데이터 범위/권한?
5. PoC 범위 (인원, 기간, 예산)?

#### [3] 제안 발표 (20분)

##### 3.1 문제점 제시 (2분)

**핵심 메시지**:
```
"현재 의원 한 분이 5대의 PC를 사용하시면서
자료 동기화도 어렵고, 망분리 때문에 AI 활용도 불가능합니다.
저희 제안은 이 두 가지 문제를 동시에 해결합니다."
```

##### 3.2 솔루션 제시 (5분)

**핵심 메시지**:
```
"VDI + 내부망 LLM + OpenClaw 조합으로,
망분리를 유지하면서 AI 비서 서비스를 제공합니다.

어디서든 VDI에 접속하면 동일한 환경에서
AI 비서를 활용한 업무가 가능합니다.

데이터는 절대 외부로 나가지 않습니다.
VDI는 화면만 전송하고, 모든 처리는 내부망에서 이루어집니다."
```

**인포그래픽 활용**:
- 데이터 흐름도 보여주며 설명
- "화면만 전송, 데이터는 내부 유지" 강조

##### 3.3 제안 옵션 (8분)

**핵심 메시지**:
```
"3가지 옵션을 준비했습니다.

옵션 A는 온프레미스로, 보안이 가장 높지만
초기 비용 9.5억이 필요합니다.

옵션 B는 클라우드로, 빠르게 시작 가능하지만
월 운영비가 4.5천만원 발생합니다.

저희가 권장하는 옵션 C는 하이브리드로,
5천만원으로 10명 PoC를 시작하고
검증 후 옵션 A나 B로 전환하는 방식입니다.

최소 비용으로 검증하고, 리스크를 줄이는 접근입니다."
```

##### 3.4 보안 아키텍처 (3분)

**핵심 메시지**:
```
"보안은 5계층으로 설계했습니다.

가장 중요한 것은 DLP, 데이터 유출 방지입니다.
클립보드 복사, 파일 다운로드, USB 모두 차단됩니다.
화면 캡처도 워터마크가 적용됩니다.

모든 LLM 입출력은 로깅되어 추적 가능합니다."
```

##### 3.5 PoC 계획 (2분)

**핵심 메시지**:
```
"PoC는 3단계로 진행합니다.

Phase 1: 10명, 4주, 5천만원으로 기능 검증
Phase 2: 50명, 6주, 1.5억으로 성능 검증
Phase 3: 100명, 8주, 2억으로 안정화 및 보안 감사

각 단계마다 Go/No-Go 기준을 두어
리스크를 최소화합니다."
```
