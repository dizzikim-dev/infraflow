## 필수 확인 질문 (5개 핵심)

### Q1. 내부망 LLM 현황

```
질문: 현재 내부망에 구축된 LLM은 어디에, 어떤 형태로 운영 중인가요?

확인 항목:
□ 위치: 온프레미스 / 프라이빗 클라우드
□ 모델: Llama / Mistral / 기타
□ 인프라: GPU 사양, 서버 현황
□ API: 제공 여부, 형식

왜 중요한가:
→ LLM 연동 방식 결정
→ 추가 투자 필요 여부 판단
```

### Q2. 외부 접근 정책

```
질문: 의원이 외부에서 내부 업무자원 접근 시 VPN/2FA/DID 적용 범위는?

확인 항목:
□ 현재 외부 접근 방식: VPN 사용 여부
□ 인증 방식: ID/PW only? 2FA? DID?
□ 접근 허용 범위: 장소/기기/시간 제한
□ 모바일 신분증 도입 현황

왜 중요한가:
→ 인증 아키텍처 설계
→ DID 연동 가능 여부
```

### Q3. VDI 보안 정책

```
질문: VDI 환경에서 클립보드/파일반출/캡처 정책을 어느 수준으로 해야 하나요?

확인 항목:
□ 클립보드: 완전 차단 / 일부 허용
□ 파일 반출: 완전 차단 / 승인 후 허용
□ 화면 캡처: 차단 / 워터마크
□ 인쇄: 차단 / 워터마크
□ USB: 차단 / 허용

왜 중요한가:
→ DLP 정책 수준 결정
→ 사용자 편의성 vs 보안 균형
```

### Q4. OpenClaw 접근 데이터

```
질문: AI 비서가 접근해야 할 1차 데이터와 권한 체계는 어떻게 되나요?

확인 항목:
□ 데이터 종류:
  - 의정포털
  - 회의록
  - 조례/규칙
  - 의안/청원
  - 일정
  - 민원
□ 권한 체계:
  - 의원 / 보좌진 / 사무처 구분
  - 공개 / 비공개 문서 분류
□ 데이터 형식: DB / 파일 / API

왜 중요한가:
→ RAG 데이터 범위 결정
→ 권한 설계
```

### Q5. PoC 범위

```
질문: PoC 목표와 범위는 어떻게 설정하면 좋을까요?

확인 항목:
□ 대상 인원: 10명 / 20명 / 더 많이?
□ 대상 선정: 의원 위주 / 보좌진 포함?
□ 기간: 1개월 / 2개월 / 3개월?
□ 예산 범위: 확보 여부
□ 성공 기준: 어떤 것을 검증?
□ 평가 방법: 설문 / 데이터 분석

왜 중요한가:
→ Phase 1 구체화
→ 리소스 계획
```

## 예상 Q&A

### Q: OpenClaw가 환각(hallucination)을 일으키면?
```
A:
• 모든 응답에 출처 표시 (RAG 기반)
• "AI 생성 내용입니다. 검증이 필요합니다" 고지
• 사용자 피드백 수집으로 지속 개선
• 중요 의사결정 전 원본 문서 확인 권장
```

### Q: 보안 사고가 발생하면?
```
A:
• 모든 행위 로깅으로 추적 가능
• DLP로 데이터 반출 원천 차단
• 세션 단위 격리로 피해 최소화
• 침투테스트로 사전 취약점 점검
```

### Q: 기존 업무 시스템과 연동은?
```
A:
• 의정포털: API 연동 또는 RAG 수집
• 일정 시스템: 캘린더 연동 가능
• 메일: 연동 범위 협의 필요
• 단계적 연동으로 리스크 최소화
```

### Q: 비용이 너무 높지 않나?
```
A:
• 옵션 C (하이브리드)로 최소 비용 시작
• PoC 5천만원으로 검증 후 판단
• 장기적으로 의원 생산성 향상 ROI
• 단계별 투자로 리스크 분산
```
