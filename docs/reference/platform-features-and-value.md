# InfraFlow — 플랫폼 기능 및 가치 정의서

> 작성일: 2026-02-25 | 버전: 1.0

---

## 1. 플랫폼 정의

**InfraFlow**는 자연어 기반 AI 인프라 컨설팅 플랫폼이다.

사용자가 "3-tier 웹 아키텍처에 WAF와 로드밸런서를 넣어줘"라고 입력하면, 인터랙티브 인프라 다이어그램을 생성하고, 왜 그 구성요소들이 그렇게 연결되는지 설명하며, 요구사항에 맞는 실제 벤더 제품을 추천한다.

### 핵심 문제 인식

인프라는 너무 복잡하다. **아무도 전체 그림을 보지 못한다.**

| 이해관계자 | 현실 | 결과 |
|-----------|------|------|
| **벤더** | 자사 제품만 안다 | Cisco는 Cisco만, Fortinet은 Fortinet만 이야기한다 |
| **고객** | 무엇이 필요한지 모른다 | 벤더가 밀어주는 제품을 산다 |
| **SI** | 다중 벤더 복잡성 | 통합 관점 부재, 프로젝트 지연 |
| **주니어 엔지니어** | 학습 곡선이 가파르다 | 단편적 지식, 아키텍처 이해 부족 |

### 해법: 3계층 가치 구조

```
┌──────────────────────────────────────────────────────────────┐
│  Layer 3: RECOMMEND (추천)                                     │
│  "이 방화벽 역할에는 Fortinet FortiGate 600F 또는              │
│   Cisco Firepower 2100을 추천합니다 — 이유는 이렇습니다"        │
│  온톨로지 + 벤더 카탈로그 → 제품 매칭 & 컨설팅                   │
├──────────────────────────────────────────────────────────────┤
│  Layer 2: UNDERSTAND (이해)                                    │
│  "WAF는 로드밸런서와 웹서버 사이에 위치합니다.                    │
│   SSL 종단 이후 HTTP 트래픽을 검사해야 하기 때문입니다"           │
│  온톨로지 기반 — 구성요소가 '왜' 그렇게 연결되는지 설명           │
├──────────────────────────────────────────────────────────────┤
│  Layer 1: VISUALIZE (시각화)                                   │
│  "3-tier 아키텍처에 보안 존을 보여줘"                            │
│  자연어 → 인터랙티브 애니메이션 다이어그램                        │
└──────────────────────────────────────────────────────────────┘
```

## 2. 핵심 기능 (Features)

### 2.1 자연어 인프라 설계 (Layer 1: Visualize)

**한 줄로 인프라를 그린다.**

- **자연어 파서**: 한국어/영어 프롬프트를 구조화된 인프라 스펙으로 변환
- **78종 인프라 노드**: 10개 카테고리(보안·네트워크·컴퓨트·클라우드·스토리지·인증·텔레콤·WAN·AI컴퓨트·AI서비스)
- **스마트 프롬프트 라우터**: 3단계 신뢰도 라우팅
  - 신뢰도 ≥ 0.9 → 템플릿 직접 매칭
  - 0.5 ~ 0.89 → Haiku 기반 시맨틱 검증
  - < 0.5 → LLM 직접 생성
- **5가지 데이터 흐름 애니메이션**: 요청(파랑), 응답(초록), 차단(빨강), 암호화(보라), 동기화(주황)
- **자동 레이아웃 엔진**: 노드 배치, 존 그룹핑, 계층 정렬
- **41개 아키텍처 템플릿**: 3-tier, DMZ, 마이크로서비스, Zero-Trust, 텔레콤, K8s, AI 파이프라인 등

**구체적 수치:**

| 항목 | 수량 |
|------|------|
| 인프라 노드 타입 | 78종 |
| 카테고리 | 10개 |
| 아키텍처 템플릿 | 41개 (32 클래식 + 9 AI) |
| 데이터 흐름 타입 | 8종 (request, response, blocked, encrypted, sync, inference, model-sync, embedding) |
| 파서 패턴 | 한국어/영어 이중 언어 |

### 2.2 온톨로지 기반 지식 그래프 (Layer 2: Understand)

**"왜 이렇게 연결되는가"를 설명할 수 있는 유일한 플랫폼.**

인프라 구성요소 간의 관계, 패턴, 안티패턴, 장애 시나리오를 구조화된 온톨로지로 관리한다.

| 지식 유형 | 수량 | 설명 |
|-----------|------|------|
| **관계(Relationships)** | 140개 | 구성요소 간 연결 방식, 프로토콜, 방향, 강도 |
| **아키텍처 패턴** | 41개 | 검증된 설계 — 필수/선택 구성요소, WAF 5대 축 점수 |
| **안티패턴** | 45개 | 알려진 나쁜 설계 — 탐지 로직 + 심각도 + 개선 방안 |
| **장애 시나리오** | 64개 | 근본 원인 분석, 복구 절차, 영향 분석 |
| **성능 프로파일** | 42개 | 처리량, 지연시간, 확장 한계 |
| **취약점** | 47개 | CVSS 등급, 보완 경로 |
| **컴플라이언스** | 6 프레임워크 | PCI-DSS, HIPAA, ISO 27001, SOC 2, GDPR, NIST 800-53 |
| **벤치마크** | 14×4 | 가격/성능/보안/확장 4차원 |
| **검증된 출처** | 40+개 | NIST, RFC, CIS, OWASP, AWS WAF, CNCF 등 |

**핵심 역량:**
- **패턴 탐지**: 사용자 다이어그램에서 알려진 아키텍처 패턴 자동 인식
- **안티패턴 경고**: 나쁜 설계를 실시간 감지하고 개선안 제시
- **갭 분석**: 7개 카테고리(누락/과잉/업그레이드/보안/컴플라이언스/성능/동반장비) 자동 분석
- **컴플라이언스 감사**: 6개 프레임워크 기준 자동 준수 여부 점검

### 2.4 RAG 파이프라인 & 추론 투명성

**AI가 "왜 이 답을 했는지" 보여주는 시스템.**

| 단계 | 기능 | 설명 |
|------|------|------|
| 1단계 | **추론 트레이스** | 5단계 파이프라인 추적 (추출→RAG→실시간보강→보강→LLM) |
| 2단계 | **사후 검증** | 온톨로지 기반 LLM 출력 검증 — 만족/누락/충돌 판정 |
| 3단계 | **그래프 유도 RAG** | BFS 인접 리스트로 관련 노드 탐색, 쿼리 확장 |
| 실시간 | **외부 데이터 보강** | maxScore < 0.5이면 GitHub/웹에서 실시간 fetch → 재검색 |

- **벡터 스토어 추상화**: ChromaDB + OpenAI ada-002 임베딩, hnswlib 폴백 확장 가능
- **LLM 프로바이더 추상화**: Claude/OpenAI 통합 인터페이스, 팩토리 패턴
- **Product Intelligence**: 36개 제품의 심층 프로파일 — 배포 방식, 통합 관계, 스케일업 경로
- **근거 패널(EvidencePanel)**: 사용자에게 AI 추론 근거, 검증 결과, 경고를 투명하게 표시

### 2.6 컨설팅 워크플로우

**7단계 요구사항 분석 → 자동 컨설팅.**

1. **조직 정보** (규모, 산업)
2. **스케일** (사용자 수, 트래픽)
3. **가용성** (SLA 요구수준)
4. **보안** (컴플라이언스 프레임워크)
5. **예산** (월 예산 범위)
6. **선호도** (벤더, 클라우드)
7. **리뷰** (종합 확인)

→ 패턴 매칭(5차원 스코어링) → 갭 분석 → 컴플라이언스 리포트 → 비용 비교표

## 4. 플랫폼의 의의와 가치

### 4.1 산업적 의의 — "인프라 지식의 민주화"

**인프라 지식은 지금까지 소수 시니어 엔지니어의 머릿속에만 존재했다.** 그들이 은퇴하면 지식도 사라진다. InfraFlow는 이 암묵지(暗默知)를 구조화된 온톨로지로 코드화하여, 누구나 접근할 수 있는 형태로 보존한다.

- **140개 관계**: "왜 WAF는 로드밸런서 뒤에 오는가"를 데이터로 설명
- **41개 패턴**: "검증된 아키텍처란 무엇인가"를 정량화
- **45개 안티패턴**: "무엇을 하면 안 되는가"를 사전 경고
- **64개 장애 시나리오**: "무엇이 잘못될 수 있는가"를 미리 학습

이것은 단순한 도구가 아니라 **인프라 지식 체계(Infrastructure Knowledge System)**이다.

### 4.2 기술적 의의 — "AI + 온톨로지의 결합"

기존 인프라 도구들은 두 가지 중 하나다:
- **그리기 도구** (Visio, Lucidchart, draw.io): 예쁘지만 "의미"가 없다
- **IaC 도구** (Terraform, Pulumi): 강력하지만 "이해"가 없다

InfraFlow는 **제3의 카테고리**를 연다:

| 기존 도구 | InfraFlow |
|-----------|-----------|
| 사용자가 모든 것을 알아야 함 | 자연어로 설명하면 AI가 설계 |
| 정적 다이어그램 | 애니메이션 데이터 흐름 |
| 그리기만 가능 | 설계 + 이해 + 추천 |
| 벤더 중립 (정보 없음) | 1,100+ 실제 제품 데이터 연동 |
| 단일 출력 | Terraform/K8s/PlantUML/PDF 멀티 내보내기 |

핵심은 **온톨로지와 LLM의 결합**이다. LLM만으로는 할루시네이션 위험이 있다. 온톨로지만으로는 유연성이 부족하다. InfraFlow는 온톨로지로 사실을 보장하고, LLM으로 유연성을 확보하며, RAG로 최신 정보를 보강하고, 사후 검증으로 출력의 정확성을 확인한다.

```
사용자 질의
  ↓
스마트 라우터 (신뢰도 기반 3단계)
  ↓
RAG 검색 (벡터 DB + 그래프 탐색)
  ↓
지식 보강 (온톨로지 컨텍스트 주입)
  ↓
LLM 생성 (Claude / OpenAI)
  ↓
사후 검증 (온톨로지 기반 정합성 체크)
  ↓
근거 표시 (트레이스 + 검증 결과)
```

### 4.3 비즈니스 가치 — "시간과 비용의 절약"

| 전통적 방식 | InfraFlow |
|------------|-----------|
| 아키텍처 설계에 수일~수주 | 자연어 입력 후 수초 |
| 벤더 미팅 수회 | 1,100+ 제품 즉시 비교 |
| 컴플라이언스 감사에 수개월 | 6개 프레임워크 자동 점검 |
| 제안서 작성에 수일 | Terraform/K8s 코드 즉시 생성 |
| 비용 산정에 수일 | 다중 벤더 비용 자동 비교 |

### 4.4 교육적 가치 — "배움의 플랫폼"

InfraFlow는 **인프라 교육 도구**로서의 가치도 가진다:
- 주니어 엔지니어가 아키텍처 패턴을 인터랙티브하게 탐색
- 안티패턴 경고를 통해 "왜 이렇게 하면 안 되는지" 실시간 학습
- 장애 시나리오를 통해 "무엇이 잘못될 수 있는지" 사전 이해
- 140개 관계 정보를 통해 "왜 이 구성요소가 여기에 있는지" 근거 확인

### 4.5 차별화 요소 — "왜 InfraFlow인가"

| 차별점 | 설명 |
|--------|------|
| **온톨로지 기반** | 단순 키워드가 아닌, 구조화된 지식 체계로 인프라 관계를 이해 |
| **추론 투명성** | AI가 왜 이 답을 했는지 5단계 트레이스로 증명 |
| **사후 검증** | LLM 출력을 온톨로지에 대조하여 정합성 자동 확인 |
| **실제 제품 데이터** | 22개 벤더 1,100+ 제품, 111개 클라우드 서비스, 39개 AI 제품 |
| **코드 생성** | 다이어그램 → Terraform/K8s 코드 직접 내보내기 |
| **이중 언어** | 한국어/영어 완전 지원 |
| **그래프 유도 RAG** | 온톨로지 간선을 따라 쿼리를 확장하는 독자적 검색 방식 |

## 5. 대상 사용자와 가치

| 사용자 | 현재 고통 | InfraFlow가 주는 가치 |
|--------|----------|---------------------|
| **인프라 엔지니어** | 이해관계자에게 전체 아키텍처를 설명하기 어렵다 | 자연어 → 비주얼 다이어그램으로 즉시 소통 |
| **프리세일즈 / 솔루션 아키텍트** | 제안서 작성에 수일이 걸린다 | 수분 만에 아키텍처 생성 + 제품 추천 + 비용 산정 |
| **IT 관리자 / CISO** | 사고 있는 것인지 모르겠다 | 제품이 어떻게 맞물리는지 시각화 + 컴플라이언스 자동 점검 |
| **SI / 시스템 통합업체** | 다중 벤더 복잡성, 통합 관점 부재 | 온톨로지가 크로스벤더 상호연결을 매핑 |
| **학생 / 주니어** | 인프라 학습 곡선이 가파르다 | 인터랙티브 다이어그램으로 탐색하며 학습 |
| **컨설턴트** | 요구사항 분석에 시간이 많이 든다 | 7단계 위자드 → 자동 갭 분석 + 컴플라이언스 리포트 |

## 7. 요약

InfraFlow는 **"인프라를 이해하는 AI"**다.

기존의 도구들이 "그리기"에 머물렀다면, InfraFlow는 **시각화(Visualize) → 이해(Understand) → 추천(Recommend)**의 3계층 가치를 통해 인프라 설계의 전체 라이프사이클을 지원한다.

140개의 온톨로지 관계, 22개 벤더의 1,100개 이상 제품 데이터, 111개 클라우드 서비스, 6개 컴플라이언스 프레임워크, 그리고 AI 추론의 투명성까지 — 이 모든 것이 자연어 한 줄에서 시작된다.

**"3-tier 웹 아키텍처를 만들어줘"**

이 한마디가 다이어그램이 되고, 설명이 되고, 추천이 되고, Terraform 코드가 된다. 그것이 InfraFlow의 가치다.
