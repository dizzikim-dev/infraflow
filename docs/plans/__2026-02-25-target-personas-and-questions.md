## Part 1: 타깃 페르소나 & 니즈

### P1. 스타트업 CTO / 테크리드

| 항목 | 내용 |
|------|------|
| 규모 | 5~50명, 시리즈 A~B |
| 배경 | 개발자 출신, 인프라 전문성 부족 |
| 상황 | 트래픽 급증 대비 아키텍처 설계 필요, 시간과 예산 제한 |
| 핵심 니즈 | "우리 서비스에 맞는 인프라를 빠르게 설계하고 비용을 예측하고 싶다" |
| 기대 가치 | 컨설턴트 없이 1시간 내 아키텍처 초안 + 비용 견적 |
| 주요 사용 기능 | 자연어 입력, 클라우드 추천, 비용 비교, Terraform 내보내기 |

### P2. SI/IT 컨설턴트

| 항목 | 내용 |
|------|------|
| 규모 | SI업체 또는 프리랜서 컨설턴트 |
| 배경 | 인프라 설계 경력 5년+, 다수 고객 대응 |
| 상황 | 고객 제안서에 아키텍처 다이어그램 + 벤더 비교표 필요 |
| 핵심 니즈 | "고객별 요구사항에 맞는 제안서를 빠르게 뽑아내고 싶다" |
| 기대 가치 | 제안서 작성 시간 70% 단축, 벤더 중립적 비교 데이터 |
| 주요 사용 기능 | 패턴 매칭, 벤더 추천, 컴플라이언스 감사, PDF 리포트 |

### P3. 중소기업 IT 관리자

| 항목 | 내용 |
|------|------|
| 규모 | 50~300명, IT팀 1~3명 |
| 배경 | 올라운더, 서버/네트워크/보안 혼자 관리 |
| 상황 | 레거시 인프라 운영 중, 클라우드 전환 검토 |
| 핵심 니즈 | "현재 인프라의 문제점을 진단하고 개선 방향을 알고 싶다" |
| 기대 가치 | 안티패턴 자동 감지, 보안 취약점 식별, 마이그레이션 로드맵 |
| 주요 사용 기능 | 다이어그램 입력, Gap 분석, 보안 감사, 클라우드 매칭 |

### P4. 대기업 인프라/클라우드 아키텍트

| 항목 | 내용 |
|------|------|
| 규모 | 1000명+, 전담 인프라팀 10명+ |
| 배경 | 특정 벤더 전문가, 멀티벤더 환경 운영 |
| 상황 | 복잡한 하이브리드/멀티클라우드 환경, 컴플라이언스 필수 |
| 핵심 니즈 | "멀티클라우드 아키텍처를 표준화하고 컴플라이언스를 자동 검증하고 싶다" |
| 기대 가치 | 벤더 간 호환성 검증, 컴플라이언스 자동 감사, 아키텍처 표준화 |
| 주요 사용 기능 | 패턴 검증, 컴플라이언스 리포트, 벤더 비교, 비용 최적화 |

### P5. 보안 엔지니어 / CISO

| 항목 | 내용 |
|------|------|
| 규모 | 중견~대기업 보안팀 |
| 배경 | 보안 전문가, 인프라 아키텍처 설계 참여 |
| 상황 | 인프라 보안 검토 요청 빈번, 컴플라이언스 감사 주기적 |
| 핵심 니즈 | "인프라 설계 단계에서 보안 취약점을 사전에 잡고 싶다" |
| 기대 가치 | 설계 시점 보안 검증, 6대 프레임워크 자동 감사, 안티패턴 탐지 |
| 주요 사용 기능 | 보안 감사, 컴플라이언스 리포트, 안티패턴 경고, 취약점 DB |

### P6. AI/ML 엔지니어

| 항목 | 내용 |
|------|------|
| 규모 | AI 스타트업 또는 대기업 AI팀 |
| 배경 | 모델 개발 전문, 인프라 운영 경험 제한적 |
| 상황 | GPU 서버 구성, MLOps 파이프라인 설계, 비용 폭증 |
| 핵심 니즈 | "AI 워크로드에 맞는 인프라를 효율적으로 설계하고 싶다" |
| 기대 가치 | AI 전용 아키텍처 패턴, GPU/추론 서버 구성, MLOps 도구 추천 |
| 주요 사용 기능 | AI 노드 타입, AI 패턴 템플릿, AI 소프트웨어 카탈로그, 비용 비교 |

### P7. DevOps / SRE 엔지니어

| 항목 | 내용 |
|------|------|
| 규모 | 중견~대기업 DevOps팀 |
| 배경 | IaC, CI/CD, 모니터링 전문 |
| 상황 | 인프라 변경 빈번, 아키텍처 문서화 부재, 장애 대응 |
| 핵심 니즈 | "인프라 현황을 시각화하고 IaC 코드를 자동 생성하고 싶다" |
| 기대 가치 | 다이어그램 → Terraform/K8s 자동 변환, 아키텍처 문서 자동화 |
| 주요 사용 기능 | Terraform 내보내기, K8s 내보내기, PlantUML, 패턴 검증 |

### P8. IT 교육자 / 학생

| 핵심 니즈 | "인프라 패턴을 눈으로 보고 이해시키고 싶다" |
| 기대 가치 | 패턴별 시각화, 안티패턴 대비 학습, 실시간 플로우 애니메이션 |
| 주요 사용 기능 | 자연어 입력, 패턴 라이브러리, 애니메이션, 안티패턴 학습 |

### P9. 공공기관 IT 담당자

| 항목 | 내용 |
|------|------|

| 기대 가치 | 컴플라이언스 자동 검증, 감사 리포트, 벤더 비교 근거 자료 |
| 주요 사용 기능 | 컴플라이언스 감사, PDF 리포트, 비용 비교, 보안 패턴 |

### P10. MSP(Managed Service Provider) / 리셀러

| 항목 | 내용 |
|------|------|
| 규모 | IT 운영 대행사, 10~50개 고객 관리 |
| 배경 | 다수 고객의 이기종 인프라 관리 |
| 상황 | 고객별 인프라 현황 파악 어려움, 표준화 필요 |
| 핵심 니즈 | "고객별 인프라를 표준화하고 효율적으로 관리/제안하고 싶다" |
| 기대 가치 | 고객별 아키텍처 템플릿, 일괄 벤더 비교, 제안서 자동화 |
| 주요 사용 기능 | 템플릿 매칭, 벤더 추천, 비용 비교, 리포트 생성 |

## Part 4: 플랫폼 기능 매핑

각 질문이 InfraFlow의 어떤 기능 레이어에서 처리되는지:

| 레이어 | 해당 질문 수 | 주요 기능 |
|--------|-------------|-----------|
| **L1: Visualize** | ~30 | 자연어 파싱, 다이어그램 생성, 애니메이션 |
| **L2: Understand** | ~35 | 패턴 매칭, 안티패턴 감지, 지식 그래프, RAG |
| **L3: Recommend** | ~35 | 벤더 추천, 클라우드 매칭, 비용 비교, 컴플라이언스 |

> **핵심 인사이트**: 가장 빈번한 니즈는 "설계 + 검증"(L1+L2)이지만, 가장 높은 지불 의향은 "추천 + 비교"(L3)에서 발생합니다. 컨설팅 가치가 곧 수익 모델입니다.
