# Meeting Agent (미팅 준비 에이전트)

## 역할
미팅 아젠다, 질문 리스트, 브리핑 자료, 회의록 정리를 담당합니다.

## 전문 영역
- 미팅 아젠다 설계
- 질문/확인사항 리스트
- 브리핑 자료 준비
- 회의록 정리
- 후속 조치 관리

## 사용 시점
- 고객 미팅을 준비할 때
- 질문/확인 사항을 정리할 때
- 회의 후 내용을 정리할 때
- 후속 조치를 추적할 때

## 프롬프트 템플릿

### 기본 템플릿
```
[Meeting Agent]

## 미팅 정보
- 일시: {YYYY-MM-DD HH:MM}
- 장소: {장소}
- 참석자: {고객측/제안사측}

## 미팅 목적
{제안 발표/요구사항 확인/PoC 리뷰 등}

## 준비 사항
- 확인할 질문: {리스트}
- 전달할 내용: {리스트}
- 준비할 자료: {리스트}

## 요청 출력물
- [ ] 아젠다
- [ ] 질문 리스트
- [ ] 1페이지 브리핑
```

### 예시: 2/13 경기도의회 미팅 준비
```
[Meeting Agent]

## 미팅 정보
- 일시: 2026-02-13 (금)
- 장소: 경기도의회
- 참석자: 의회 담당자, KT 제안팀

## 미팅 목적
VDI 기반 의원 업무환경 통합 + 내부망 LLM 활용 방안 제안

## 준비 사항
- 확인할 질문: 내부망 LLM 현황, VDI 정책, 보안 요구사항
- 전달할 내용: OpenClaw + VDI 아키텍처, PoC 제안
- 준비할 자료: 인포그래픽, 아키텍처 다이어그램

## 요청 출력물
- [x] 미팅 아젠다
- [x] 필수 확인 질문 리스트
- [x] 1페이지 브리핑
```

- 자유 질의

## 5. 다음 단계 (5분)
- PoC 범위 협의
- 일정 조율
- 담당자 지정

## 질문 리스트 템플릿

### 카테고리별 질문

#### 인프라 현황
1. 내부망 LLM은 어디에, 어떤 형태로 운영 중인가요?
   - 온프레미스 / 프라이빗 클라우드
   - 모델 종류 (Llama, Mistral 등)
   - GPU 인프라 현황

2. 현재 VDI 또는 가상화 환경이 있나요?
   - 사용 중인 VDI 솔루션
   - 사용자 규모
   - Docker 사용 가능 여부

#### 보안 정책
3. 의원이 외부에서 내부 자원 접근 시 허용 범위는?
   - 모바일 VPN 허용 여부
   - 2FA/DID 적용 현황
   - 접근 허용 장소/기기/시간

4. VDI에서 클립보드/파일반출/캡처 정책은?
   - 현재 정책 현황
   - 조정 가능 범위

#### 데이터/권한
5. OpenClaw가 접근해야 할 데이터는?
   - 의정포털, 민원, 일정, 회의자료, 회의록
   - 권한 체계 (의원/보좌진/사무처)
   - 민감 데이터 분류

#### PoC 관련
6. PoC 목표/범위/기간은?
   - 파일럿 대상 인원 (10명? 50명?)
   - 검증하고자 하는 핵심 기능
   - 성공 기준

## 브리핑 자료 템플릿

**미팅**: 경기도의회 VDI + AI 비서 제안
**일시**: 2026-02-13 (금)

---

## 핵심 메시지 (3줄 요약)
1. **구현 가능**: OpenClaw + VDI + 내부망 LLM 조합은 기술적으로 구현 가능
2. **보안 유지**: 망분리 유지하면서 AI 서비스 제공 가능
3. **단계적 도입**: 10명 → 50명 → 155명 단계별 확대 권장

| VDI | 가상 데스크톱 | DMZ |
| OpenClaw | AI 에이전트 | VDI 내 Docker |
