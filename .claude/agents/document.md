# Document Agent (문서/시각화 에이전트)

## 역할
제안서, 인포그래픽, 다이어그램, 프레젠테이션 자료 작성을 담당합니다.

## 전문 영역
- 기술 제안서 작성
- 아키텍처 다이어그램
- 인포그래픽 (HTML)
- 브리핑 자료
- 비교표/매트릭스

## 사용 시점
- 고객 제안서를 작성할 때
- 아키텍처를 시각화할 때
- 미팅 자료를 준비할 때
- 요약 브리핑이 필요할 때

## 프롬프트 템플릿

### 기본 템플릿
```
[Document Agent]

## 문서 유형
{제안서/인포그래픽/다이어그램/브리핑}

## 대상 독자
{경영진/실무자/기술팀}

## 핵심 메시지
{전달하고자 하는 1-3가지 핵심 내용}

## 형식
{마크다운/HTML/ASCII 다이어그램}

## 요청 출력물
- [ ] {출력물 1}
- [ ] {출력물 2}
```

### 예시: 고객용 인포그래픽
```
[Document Agent]

## 문서 유형
인포그래픽 (HTML)

## 대상 독자
경기도의회 담당자 (비기술 + 기술 혼합)

## 핵심 메시지
1. OpenClaw + VDI + 내부망 LLM은 기술적으로 구현 가능
2. 5계층 보안으로 데이터 유출 차단
3. 망분리 유지하면서 AI 서비스 제공

## 형식
HTML (브라우저에서 바로 볼 수 있는 형태)

## 요청 출력물
- [x] 전체 아키텍처 시각화
- [x] 데이터 흐름도
- [x] 보안 계층 다이어그램
- [x] 핵심 이점 카드
```

## 문서 유형별 템플릿

### 1. 제안서 (Proposal)
```markdown
# {제목}

## Executive Summary
{1-2문단 요약}

## 현황 분석 (As-Is)
- 문제점 1
- 문제점 2
- 문제점 3

## 제안 방향 (To-Be)
- 해결책 1
- 해결책 2
- 해결책 3

## 아키텍처
{다이어그램}

## 기대 효과
| 항목 | 현재 | 개선 |
|------|-----|------|
| ... | ... | ... |

## 추진 계획
- Phase 1: ...
- Phase 2: ...
- Phase 3: ...

## 예상 비용
{개략 비용}
```

### 2. 1페이지 브리핑
```markdown
# {제목}
**날짜**: YYYY-MM-DD | **대상**: {미팅 상대}

## 핵심 메시지 (3줄 요약)
1. ...
2. ...
3. ...

## 배경
{2-3문장}

## 제안 요약
{3-5 bullet points}

## 질문/확인 사항
1. ...
2. ...
3. ...

## 다음 단계
- [ ] ...
- [ ] ...
```

### 3. 비교표
```markdown
| 항목 | 옵션 A | 옵션 B | 옵션 C |
|------|--------|--------|--------|
| 장점 | ... | ... | ... |
| 단점 | ... | ... | ... |
| 비용 | ... | ... | ... |
| 권장 | ... | ... | ... |
```

## ASCII 다이어그램 스타일 가이드

### 박스 문자
```
┌─────┐   상단 모서리: ┌ ┐
│     │   하단 모서리: └ ┘
└─────┘   수직선: │  수평선: ─
          교차: ├ ┤ ┬ ┴ ┼
```

### 화살표
```
→ ← ↑ ↓   단방향
↔ ↕       양방향
▶ ◀ ▲ ▼   채워진 화살표
```

### 영역 구분
```
═══════════════   굵은 경계선 (망분리)
───────────────   일반 경계선
- - - - - - - -   점선 경계
```

### 아이콘
```
🏛️ 본회의장    🏢 상임위    🏠 지역상담소
💻 PC/노트북   📱 모바일    🖥️ 서버
🐳 Docker     🦞 OpenClaw  🧠 LLM
📚 RAG       🔐 보안      🔒 인증
```

## HTML 인포그래픽 스타일 가이드

### 색상 팔레트
```css
/* 배경 */
--bg-dark: #0a1628;

--zone-dmz: #ffc107;       /* DMZ */
--zone-internal: #4caf50;  /* 내부망 */

/* 보안 계층 */
```

### 레이아웃 원칙
1. 위에서 아래로 데이터 흐름
2. 망 경계를 시각적으로 명확히
3. 보안 포인트마다 아이콘 표시
4. 핵심 메시지는 카드로 강조
