# Dispatching Parallel Agents Skill

## Overview
복잡한 작업을 여러 에이전트에 분배하여 병렬로 처리합니다.

## Activation
- `/parallel [작업들]` 명령으로 활성화
- 독립적인 여러 작업 감지 시 자동 제안

## When to Use

### 적합한 경우
```
✅ Frontend + Backend 동시 개발
✅ 여러 컴포넌트 독립적 구현
✅ 테스트와 문서화 동시 진행
✅ 다양한 파일 형식 변환
✅ 병렬 리서치/탐색
```

### 부적합한 경우
```
❌ 순차적 의존성이 있는 작업
❌ 공유 상태를 수정하는 작업
❌ 하나의 파일을 동시에 수정
❌ 결과 통합이 복잡한 작업
```

## Process

### Step 1: 작업 분석
```
목표: 병렬화 가능한 작업 식별

질문:
□ 각 작업이 독립적인가?
□ 공유 리소스가 있는가?
□ 순서 의존성이 있는가?
□ 결과 통합 방법은?
```

### Step 2: 작업 분배
```
템플릿:

┌─────────────────────────────────────────────┐
│              메인 작업 컨텍스트              │
├─────────────────────────────────────────────┤
│                                             │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│   │ Agent A  │  │ Agent B  │  │ Agent C  │ │
│   │ [작업1]  │  │ [작업2]  │  │ [작업3]  │ │
│   └────┬─────┘  └────┬─────┘  └────┬─────┘ │
│        │             │             │        │
│        ▼             ▼             ▼        │
│   ┌──────────────────────────────────────┐ │
│   │           결과 통합 지점              │ │
│   └──────────────────────────────────────┘ │
│                                             │
└─────────────────────────────────────────────┘
```

### Step 3: 에이전트 실행
```
각 에이전트에 전달할 정보:
1. 명확한 작업 범위
2. 입력/출력 형식
3. 제약사항
4. 완료 기준
5. 다른 에이전트와의 인터페이스
```

### Step 4: 결과 통합
```
통합 전략:
□ 각 결과 검증
□ 충돌 해결
□ 일관성 확인
□ 전체 테스트
```

## InfraFlow 적용

### 병렬 스트림 예시
```
┌─────────────────────────────────────────────────────────┐
│                    새 기능 개발                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Stream A (Frontend)         Stream B (Backend)        │
│   ─────────────────          ─────────────────         │
│   ┌───────────────┐          ┌───────────────┐         │
│   │ 노드 컴포넌트  │          │ 파서 로직     │         │
│   │ 구현          │          │ 구현          │         │
│   └───────┬───────┘          └───────┬───────┘         │
│           │                          │                  │
│           ▼                          ▼                  │
│   ┌───────────────┐          ┌───────────────┐         │
│   │ 스타일링      │          │ API 엔드포인트 │         │
│   └───────┬───────┘          └───────┬───────┘         │
│           │                          │                  │
│           └──────────┬───────────────┘                  │
│                      ▼                                  │
│              ┌───────────────┐                          │
│              │ 통합 테스트    │                          │
│              └───────────────┘                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 에이전트 정의
```
Frontend Agent:
- 담당: React 컴포넌트, 스타일, 훅
- 도구: React Flow, Framer Motion, Tailwind
- 출력: 컴포넌트 파일, 테스트

Backend Agent:
- 담당: 파서, API, 데이터 로직
- 도구: TypeScript, Jest
- 출력: 라이브러리 파일, 테스트

Integration Agent:
- 담당: 통합, E2E 테스트
- 도구: Playwright
- 출력: 통합 테스트, 문서
```

## Example

```
/parallel 새로운 Firewall 노드 컴포넌트 추가

분석:
├── 작업 A: FirewallNode.tsx 컴포넌트 (Frontend)
├── 작업 B: firewall 파싱 패턴 (Backend)
├── 작업 C: firewall 테스트 케이스 (QA)
└── 의존성: 없음 (병렬 가능)

실행:
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [Agent A]                 [Agent B]     [Agent C]      │
│  FirewallNode.tsx          patterns.ts   firewall.test  │
│  ┌───────────────┐        ┌──────────┐  ┌───────────┐  │
│  │ - 아이콘      │        │ 파싱     │  │ 테스트    │  │
│  │ - 스타일     │        │ 패턴     │  │ 케이스    │  │
│  │ - 인터랙션   │        │ 추가     │  │ 작성      │  │
│  └───────────────┘        └──────────┘  └───────────┘  │
│         │                      │              │         │
│         └──────────────────────┴──────────────┘         │
│                        ▼                                │
│              ┌─────────────────────┐                    │
│              │ 통합 및 전체 테스트  │                    │
│              └─────────────────────┘                    │
│                                                         │
└─────────────────────────────────────────────────────────┘

결과:
✅ Agent A: FirewallNode.tsx 완료
✅ Agent B: patterns.ts 업데이트 완료
✅ Agent C: firewall.test.ts 완료
✅ 통합: 전체 테스트 통과
```

## Rules

### MUST DO
- 작업 독립성 확인
- 명확한 인터페이스 정의
- 결과 통합 계획 수립
- 충돌 해결 전략 준비

### MUST NOT
- 의존적 작업 병렬화
- 같은 파일 동시 수정
- 통합 테스트 생략
- 에이전트 간 통신 복잡화
