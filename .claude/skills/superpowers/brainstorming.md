# Brainstorming Skill

## Overview
소크라테스식 대화를 통해 설계를 정제하고 검증합니다.

## Activation
- `/brainstorm [주제]` 명령으로 활성화
- 새로운 기능 설계 시 자동 제안

## Process

### Phase 1: 요구사항 탐색
```
목표: 진짜 필요한 것이 무엇인지 파악

질문:
□ 이 기능이 해결하려는 문제는 무엇인가?
□ 누가 이 기능을 사용하는가?
□ 성공적인 결과는 어떤 모습인가?
□ 이 기능 없이는 어떤 문제가 있는가?
□ 비슷한 기존 솔루션이 있는가?
```

### Phase 2: 제약사항 파악
```
목표: 기술적/비즈니스적 제약 확인

질문:
□ 반드시 지켜야 할 기술적 제약은?
□ 시간/리소스 제약은?
□ 기존 시스템과의 호환성은?
□ 성능 요구사항은?
□ 보안 요구사항은?
```

### Phase 3: 대안 탐색
```
목표: 여러 접근 방식 비교

구조:
Option A: [첫 번째 접근]
├── 장점: ...
├── 단점: ...
└── 리스크: ...

Option B: [두 번째 접근]
├── 장점: ...
├── 단점: ...
└── 리스크: ...

Option C: [세 번째 접근]
├── 장점: ...
├── 단점: ...
└── 리스크: ...
```

### Phase 4: 결정 및 문서화
```
목표: 최종 결정 및 근거 기록

결정 템플릿:
## Decision: [결정 사항]

### Context
[왜 이 결정이 필요한가]

### Options Considered
1. [Option A] - 선택하지 않은 이유
2. [Option B] - 선택한 이유 ✓
3. [Option C] - 선택하지 않은 이유

### Consequences
- 긍정적: ...
- 부정적: ...
- 리스크: ...

### Action Items
- [ ] ...
- [ ] ...
```

## InfraFlow 적용

### 브레인스토밍 주제 예시
```
1. 새로운 인프라 컴포넌트 추가
   - 어떤 카테고리에 속하는가?
   - 기존 컴포넌트와의 관계는?
   - 시각적 표현은?

2. 애니메이션 시나리오 설계
   - 어떤 흐름을 보여줄 것인가?
   - 타이밍은 어떻게?
   - 사용자 인터랙션은?

3. LLM 프롬프트 패턴 설계
   - 어떤 표현을 인식할 것인가?
   - 모호한 입력은 어떻게 처리?
   - 에러 처리는?
```

### Pessimist + Optimist 연동
```
브레인스토밍 결과물에 대해:

@Pessimist: 리스크 분석
- 이것이 실패한다면 가장 큰 이유는?
- 성능 병목은 어디서?
- 사용자가 불편해할 부분은?

@Optimist: 기회 분석
- 가장 큰 가치는?
- 확장 가능한 방향은?
- 경쟁 우위는?
```

## Example

```
/brainstorm VDI 인프라 시각화 개선

Phase 1: 요구사항 탐색
Q: 현재 VDI 시각화의 문제점은?
A: 복잡한 네트워크 구조를 한눈에 파악하기 어려움

Q: 사용자는 누구인가?
A: 인프라 관리자, 보안 담당자, 의사결정권자

Q: 성공적인 결과는?
A: 5초 안에 전체 구조 파악, 문제 지점 즉시 식별

Phase 2: 제약사항
- Next.js + React Flow 사용
- 100개 이상 노드 지원 필요
- 실시간 업데이트 (옵션)

Phase 3: 대안
Option A: 계층적 레이아웃
├── 장점: 직관적, 익숙함
├── 단점: 넓은 공간 필요
└── 리스크: 깊은 계층에서 복잡

Option B: 클러스터 기반 레이아웃 ✓
├── 장점: 그룹화 명확, 확장성
├── 단점: 학습 곡선
└── 리스크: 그룹 간 관계 표현

Option C: 자유 배치 + 자동 정렬
├── 장점: 유연성
├── 단점: 일관성 부족
└── 리스크: 혼란 가능

Phase 4: 결정
→ Option B 선택 (클러스터 기반)
→ Zone 별 그룹화 (DMZ, Internal, DB)
→ 확대/축소로 세부사항 탐색
```
